Welcome to litefunctions :)

Usage:
Install:
helm install litefunctions oci://registry-1.docker.io/ashupednekar535/litefunctions

Check status:
helm status litefunctions
kubectl get pods -n {{ .Release.Namespace }}

Docs:
Chart usage docs: https://github.com/ashupednekar/litefunctions/blob/main/chart/README.md
Full platform docs: https://ashupednekar.github.io/litefunctions/docs/

Gateway API is controlled by gatewayApi.enabled and gatewayApi.gatewayClassName.

TLS modes:
tls.certManager.enabled=true with tls.certManager.selfSigned.enabled=true: cert-manager creates a self-signed certificate.
tls.certManager.enabled=true with tls.certManager.letsencrypt.enabled=true: cert-manager creates a Let's Encrypt certificate. Set gatewayApi.tls.certManager.letsencrypt.email.
tls.certManager.enabled=false: no certificate resources are created; provide gatewayApi.tls.secretName yourself.

Local access for clusters without a real LoadBalancer address:

kubectl -n {{ .Release.Namespace }} port-forward svc/{{ include "litefunctions.gatewayName" . }}-istio 18080:8080 18443:443

curl --noproxy '*' -H 'Host: litefunctions.portal' http://127.0.0.1:18080/

curl --noproxy '*' -H 'Host: litefunctions.gitea' http://127.0.0.1:18080/

curl --noproxy '*' -k -H 'Host: litefunctions.gitea' https://127.0.0.1:18443/

If you want locally trusted certs with mkcert while using tls.certManager.enabled=false:

mkcert -install

mkcert -cert-file /tmp/litefunctions-local.pem -key-file /tmp/litefunctions-local-key.pem litefunctions.portal litefunctions.gitea

kubectl -n {{ .Release.Namespace }} create secret tls {{ .Values.gatewayApi.tls.secretName | default "litefunctions-gateway-tls" }} --cert=/tmp/litefunctions-local.pem --key=/tmp/litefunctions-local-key.pem --dry-run=client -o yaml | kubectl apply -f -
